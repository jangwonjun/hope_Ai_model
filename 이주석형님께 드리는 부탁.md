## 0. 이 문서의 목적 (형님이 해 주시면 좋은 일)

1. **`hope` 저장소** 안의 **`speech-coach`** 패키지에서 **음소(IPA) 단위 CTC 파인튜닝**을 GPU로 돌려 주시기 바랍니다.

## A. AI설계서 v0.8 기준 — 필독 (모델·토크나이저·배치·형님 역할)

이 절은 **`AI설계서_v0.8.txt` §1.3, §4, §5, §6, §7, §9** 를 기준으로 합니다.  
코드 기본값(`train_ctc.py`)과 **다른 부분은 “설계서 우선”** 으로 형님이 4080에서 맞춰 주시면 됩니다.

### A.1 우리가 만드는 것 (한 줄)

**Wav2Vec2-XLSR-base + CTC** 로 **한국어 IPA 음소**를 예측하는 **Module A (Phoneme Recognizer)** 학습.  
앱은 이 출력을 **목표 음소와 비교**해 조음 피드백을 줍니다 (§1.2 청자 모델).

### A.2 사전학습 모델 (형님이 쓸 Hub 이름)

| 항목 | AI설계서 v0.8 | Hugging Face (실제 학습 시) |
|------|----------------|-----------------------------|
| 아키텍처 | Wav2Vec2-XLSR-**base** (~95M) | `Wav2Vec2ForCTC` |
| Hub ID | 문서 예시 `wav2vec2-base-xlsr` | **`facebook/wav2vec2-xls-r-300m`** ← `configs/model/wav2vec2_base_xlsr.yaml` 와 동일 |
| 본학습 | Stage 1·2 메인 | **`--smoke` 없이** 위 모델 |
| 연결 검증만 | W2 sanity / 파이프 확인 | `--smoke` 시 **`facebook/wav2vec2-base-960h`** (가벼움, **품질용 아님**) |

**Whisper는 쓰지 않습니다** (§5.1).  
**단어 LM·beam+LM은 Stage 1에서 쓰지 않습니다** (§5.4.3 — LM bias가 과제에 해로움).

### A.3 토크나이저·어휘 (IPA)

| 항목 | 설계서 | 코드 구현 |
|------|--------|-----------|
| 종류 | CTC phoneme tokenizer | **`Wav2Vec2CTCTokenizer`** (`speech_coach/data/ipa_vocab.py`) |
| 어휘 | **한국어 IPA 35토큰** (§5.2) | `PHONEME_ORDER` 35개 + CTC blank = **`<pad>` id 0** |
| 다글자 음소 | `tɕ`, `t͈`, `kʰ` 등 **한 토큰** | `vocab.json` 에 동일 |
| 라벨 생성 | g2pK + 150단어 lookup (§5.3) | **`g2p_ko.g2p_sentence(한글 전사)`** → manifest의 `transcript` |
| HF 저장 후 | — | `vocab_size` **38 근처** (`|`, `<s>`, `</s>` 추가) → **정상** |
| manifest 우선 | — | JSONL에 **`target_phonemes`** 가 있으면 g2p 대신 사용 |

**형님이 `final/` 받을 때 같이 있어야 하는 파일:**  
`vocab.json`, `tokenizer_config.json`, `processor_config.json`, `config.json`, `model.safetensors`

### A.4 학습 하이퍼파라미터 (4080, 설계서 §5.4.2 · `configs/train/stage1.yaml`)

설계서 **권장값**과 **현재 `train_ctc.py` 기본값**을 구분합니다.

| 항목 | AI설계서 v0.8 (4080 base) | `train_ctc.py` 기본 | 형님 본학습 권장 |
|------|---------------------------|---------------------|------------------|
| Optimizer | AdamW β1=0.9, β2=0.98, wd=0.005 | Trainer 기본 AdamW | 설계서 값에 가깝게 쓰려면 추후 Trainer 인자 확장; **당장은 lr 3e-5 중심** |
| Learning rate | **3e-5**, warmup **10%**, linear decay | **3e-5**, warmup_ratio **0.1** | **`--learning_rate 3e-5`** (기본과 동일) |
| per_device batch | **16** | **2** (보수적) | **OOM 없으면 16**, 안 되면 8→4 |
| gradient accumulation | **2** | **2** | **2** → **effective batch ≈ 32** (16×2) |
| Mixed precision | **bf16** (4080 Ada) | `--bf16` 플래그 필요 | **`--bf16`** (CUDA) |
| Gradient checkpointing | True | CUDA 시 True | 유지 |
| Epochs (설계) | Stage1 **20**, Stage2 **10** | **스텝 기반** (`--max_steps`) | 데이터 양 보고 **max_steps 환산** (아래 A.6) |
| CTC label smoothing | 0.1 | **미구현** | 있으면 좋음, 없어도 1차 학습 가능 |
| Feature encoder | **첫 5 epoch freeze → 이후 unfreeze** | **학습 내내 freeze** (`freeze_feature_encoder`) | 1차는 코드 그대로; **성능 부족 시** unfreeze 논의 |
| Loss | CTC mean | `ctc_loss_reduction="mean"` | 동일 |

**배치 계산 예 (설계서 목표):**

```text
effective_batch = train_batch_size × gradient_accumulation_steps × GPU개수
목표 예: 16 × 2 × 1 = 32
```

OOM 시: `train_batch_size` 를 **8 또는 4** 로 내리고, `gradient_accumulation_steps` 를 **4 또는 8** 로 올려 **effective 32** 유지.

### A.9 설계서 대비 “아직 코드에 없는 것” (형님 참고)

- `train_ctc` **학습 중 val PER 자동 로깅** (split 스크립트는 있음)  
- **Augmentation** (§6.2)  
- **5 epoch 후 feature encoder unfreeze** (§5.4.2)  
- **CTC label smoothing 0.1**  
- **Stage2 beam search** (추론 단계, §5.4.3)  
- **PER 평가** `eval_test.py` 스켈레톤  

→ **있어도 1차 학습은 가능**하고, 위 항목은 W4~W9에서 우리가 이어서 붙이거나 형님과 합의 후 반영.

### A.10 학습 시간 (설계서 §5.1.1, 4080 + XLS-R base)

| 항목 | 내용 |
|------|------|
| 설계서 참고 | **base 모델 1 Stage ≈ 3~4일** (effective batch 32, ~100h급 데이터 가정) |
| 실제 | **총 오디오 시간 × max_steps × Stage 개수**에 비례 |
| D1 train 01~05 + D2 본편 + Stage 1-A→1-B→2 | **1주일~수주** 가능 — 일정은 팀과 `max_steps` 합의 |
| 스모크(2 step) | **시간 예측용 아님** |

`max_steps` 환산 예:  
`steps_per_epoch ≈ (train 발화 수) / effective_batch` → `max_steps ≈ steps_per_epoch × 목표 epoch`

### A.11 train / val / test 분리 (설계서 §4.6) — 형님이 꼭 할 일

| 규칙 | 내용 |
|------|------|
| Speaker-disjoint | **같은 `speaker_id`가 train·val·test에 동시에 나오면 안 됨** |
| 비율 | D1·D2 각각 **80% train / 10% val / 10% test** |
| 평가 | **val**으로 학습 중 PER, **test**는 마지막에만 (튜닝에 넣지 않기) |
| 금지 | **`eval_clean` 전체를 train으로만 쓰고 “학습 완료”라고 하지 않기** (과대평가) |

## 1. 현재 상황 (프로젝트·코드·데이터)

### 1.1 프로젝트 배경

- 프로젝트 이름: **HOPE** (한국어 조음·발화 관련 AI 엔진 골격).  
- 설계 참고: 저장소 루트의 **`AI설계서_v0.8.txt`** — 특히 **§5.2 한국어 IPA 35토큰**, **§5.3 g2p** 방향.  
- **이전에는** 전사 문자(한 글자 단위)로 CTC를 돌리는 수준이었고, **지금은** 설계에 맞춰 **IPA 음소 시퀀스**로 라벨을 만들고 그에 맞는 **고정 vocab**으로 학습하도록 바꾼 상태입니다.

### 1.2 코드가 있는 위치 (형님 PC에 클론한다고 가정)

|------|------|
| 베이스 모델(본학습 권장) | `facebook/wav2vec2-xls-r-300m` |

## 5. 스모크 학습 (GPU 전에도 가능 — 연결만 확인)

**목적:** Hub 다운로드, 데이터 로딩, CTC 한두 epoch 돌고 **`final/` 저장**까지.

```bash
cd /path/to/hope/speech-coach
source .venv/bin/activate

python -m speech_coach.training.train_ctc \
  --smoke \
  --output_dir ./checkpoints/smoke-jusuk
```

`--smoke` 가 켜지면 스크립트가 **자동으로**:

- `model_name` → **`facebook/wav2vec2-base-960h`**
- `max_steps` → **최대 20**으로 캡
- `max_samples` → **최대 64**
- `train_batch_size` → **최소 1**
- `gradient_accumulation_steps` → **1**

즉, **형님이 `--max_steps 100000` 을 줘도 smoke 에선 잘립니다.** 본학습 때는 **`--smoke` 를 빼야** 합니다.

**정상 종료 시:**  
콘솔 맨 끝에 `Saved to .../final` 비슷한 문구, 폴더에 `model.safetensors`, `config.json`, `vocab.json` 등이 생깁니다.

## 6. 본학습 (형님께 부탁드리는 본체)

### 6.1 본학습 템플릿 (설계서 §5.4.2 배치 16×2 권장)

**상세·Stage 구분은 위 [A절](#a-ai설계서-v08-기준--필독-모델토크나이저배치형님-역할) 참고.**

```bash
cd /path/to/hope/speech-coach
source .venv/bin/activate

python scripts/extract_hope_data.py
python scripts/prepare_all_manifests.py --stage2-mix   # 또는 --stage1b

python -m speech_coach.training.train_ctc \
  --manifest /path/to/hope/data/manifests/stage2_d1_50_d2_50.jsonl \
  --repo_root /path/to/hope \
  --output_dir ./checkpoints/ctc-stage2-final \
  --model_name facebook/wav2vec2-xls-r-300m \
  --max_steps 50000 \
  --learning_rate 3e-5 \
  --train_batch_size 16 \
  --gradient_accumulation_steps 2 \
  --seed 42 \
  --bf16
```

- **`--smoke` 없음** → 본학습.  
- **배치:** 설계서 **per_device 16, accum 2 → effective 32**. OOM 시 batch 8·accum 4 등으로 **32 유지**.  
- **`max_steps`**: 설계 **epoch 20/10** 에 맞게 데이터 양으로 환산 (스텝 기반 CLI).  
- **`--bf16`**: 4080 CUDA 에서 켜기 (§5.4.2).

**Kspon 경로를 명시하고 싶으면:**

```bash
python -m speech_coach.training.train_ctc \
  --output_dir ./checkpoints/ctc-stage1-ipa-xlsr \
  --model_name facebook/wav2vec2-xls-r-300m \
  --kspon_trn /절대/경로/hope/data/raw/kspon/extracted/eval_clean.trn \
  --kspon_audio_root /절대/경로/hope/data/raw/kspon/extracted \
  --max_steps 50000 \
  --learning_rate 3e-5 \
  --train_batch_size 8 \
  --gradient_accumulation_steps 2 \
  --bf16
```

### 6.2 manifest 로 학습할 때

`hope` 루트를 **`--repo_root`** 로 넘기면, manifest 안의 `audio_path` 가 상대경로여도 합쳐서 찾습니다.

  --max_steps 50000 \
  --learning_rate 3e-5 \

  --gradient_accumulation_steps 4 \

**manifest JSONL 한 줄에 들어가면 좋은 필드 (요약):**

- **`audio_path`**: wav/pcm 등 (상대면 `repo_root` 기준)  
- **`transcript`**: 한글 전사 → g2p  
- 또는 **`target_phonemes`**: `["s","a","k",...]` 처럼 **§5.2 토큰만** (있으면 우선)

### 6.3 `hope_Ai_model` 보조 레포의 스크립트를 쓸 때 (선택)

## 8. 로그에서 “이상이 아닌 것” vs “진짜 문제”

### 8.1 정상으로 봐도 되는 것

- **`lm_head` weight / bias MISMATCH → Reinit**  
  - 우리 `vocab_size` 가 사전학습과 달라서 **CTC 출력층만 새로 초기화**된 것.  
- **`vocab_size` 가 38 근처**  
  - IPA 35 + HF 추가 토큰.  
- **처음 한 번 모델 다운로드가 느림**  
- **`Warning: unauthenticated requests to HF Hub`**  
  - `HF_TOKEN` 권장일 뿐, 당장 실패는 아님.

### 8.2 진짜로 손봐야 할 것

- **`Dataset is empty`**  
  - `trn` 경로·PCM 존재·manifest `audio_path` 틀림.  
- **CUDA OOM**  
  - `train_batch_size` 줄이기, `gradient_checkpointing` 은 스크립트상 CUDA 일 때 이미 켜짐 — 그래도 부족하면 배치 더 축소.  
- **`FileNotFoundError: audio not found`**  
  - `--repo_root` 또는 실제 파일 위치 불일치.
