[HOPE API 프론트 연동 안내]

🔗 Base URL
https://go-neung.activejang.com

📖 Swagger (브라우저에서 API 테스트)
https://go-neung.activejang.com/docs

✅ 서버 확인
GET /health

🎤 메인 API (녹음 1번 분석)
POST /v1/utterance/analyze
Content-Type: multipart/form-data

필수
• audio : wav 파일 (16kHz mono, 5초 이하)
• target_word : 연습 단어 (예: 사과)

선택
• target_phonemes : 비워도 됨! (비우면 target_word에서 음소 자동 변환)
• user_id : 사용자 ID

응답 주요 필드
• pcc : 점수 (0~100)
• phoneme_results : 음소별 OK/SUB/DEL/DIS
• feedback.kid_text : 아이에게 보여줄 멘트

프론트 fetch 예시
---
const form = new FormData();
form.append("audio", wavBlob, "utterance.wav");
form.append("target_word", "사과");

const res = await fetch(
  "https://go-neung.activejang.com/v1/utterance/analyze",
  { method: "POST", body: form }
);
const data = await res.json();
// data.pcc, data.phoneme_results, data.feedback.kid_text
---

Swagger 테스트 방법
1) /docs 접속
2) POST /v1/utterance/analyze → Try it out
3) audio에 wav 선택, target_word에 사과
4) target_phonemes는 비우고 "Send empty value" 체크
5) Execute

연습 단어 (1차)
사과/다과, 자두/차두, 바다/파다, 소리/도리, 가방/카방, 불/물

⚠️ 참고
• model_version=stage1b-mix 이면 실제 모델 동작 중
• apple_err wav 넣으면 PCC 100 아님 (예: 80)
• 오디오는 wav 권장 (webm은 추후)
• 상세 문서: docs/API_프론트_연동가이드.md
