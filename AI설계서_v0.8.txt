1. Executive Summary
1.1 무엇을 만드는가
8~12세 한국어 사용 아동의 발화를 입력받아 (1) 발화된 음소 시퀀스를 추출하고, (2) 목표 발화와 정렬하여 음소 단위 정/오를 판정하며, (3) 음향학적 deviation을 측정하여 정도(degree)를 표현하고, (4) PCC(자음정확도)를 산출하여 게임형 인터페이스로 피드백하는 모바일·서버 통합 AI 시스템.
1.2 시스템 정체성
본 시스템은 임상 진단 도구가 아니라 "한국어 음소를 정확히 듣는 AI 청자(listener)"이다. 사용자가 어떻게 발음하든 들리는 음향대로 음소를 출력하고, 목표 음소와 비교해 차이를 알려준다. 조음음운장애 아동의 미발음 데이터를 학습할 필요가 없으며, 정상 발화 데이터로만 학습한다.
1.3 핵심 설계 결정 (TL;DR)
	•	ASR 모델: Wav2Vec2-XLSR-base (95M, 4080 환경) + CTC head + 한국어 IPA vocab
	•	학습 데이터: KsponSpeech (성인) 50% + AIHub 어린이 음성 데이터셋 sn=266 (학년 1~6) 50%, 정상 발화만 사용
	•	음향학적 분석: OpenSMILE eGeMAPS 88-dim feature → 정상 분포 reference 구축 → z-score 기반 deviation 측정
	•	변동 분류: Rule 기반 SUB/DEL + OpenSMILE z-score 기반 DIS (ML 학습 없음)
	•	Forced alignment: torchaudio.functional.forced_align 도입
	•	평가: AIHub val PER (정량) + 라이브 시연 (정성)
	•	임상 검증·LLM 피드백: HOPE Research 후속 단계로 분리

2. 시스템 정체성 및 본질 정의
2.1 핵심 통찰
Wav2Vec2-CTC는 LM-light (Language Model 보정이 약함) 모델로, frame-level acoustic posterior를 직접 출력한다. 즉 들린 음향을 그대로 음소로 매핑할 뿐, "이 단어가 더 자연스럽다"라는 언어 정보를 거의 사용하지 않는다.
이 통찰의 자연스러운 도출:
	•	정상 ㅅ 음향 → ㅅ 출력 / 정상 ㄷ 음향 → ㄷ 출력만 정확히 학습하면 됨
	•	사용자가 "사과"를 "다과"라고 발음하면 → 모델은 들린 그대로 ㄷ 출력
	•	우리는 모델 출력을 목표 음소와 비교해서 차이 검출
결론: misarticulation 학습 데이터는 필요 없다. 정상 발화 학습만으로 misarticulation 자동 인식이 가능하다.

2.3 발표 Narrative
본 시스템은 한국어 발음의 음소 단위 차이를 분석하는 AI prototype이다. 사용자가 어떻게 발음하든 모델은 들리는 음향대로 음소를 출력하며, 이를 목표 음소와 비교해 차이를 알려준다. 학습은 정상 발화 데이터(KsponSpeech 성인 + AIHub 아동)로만 이루어졌으며, 모델은 misarticulation 데이터를 본 적 없음에도 Wav2Vec2-CTC의 LM-light 특성상 misarticulation을 자연스럽게 인식한다.
임상 환자 대상 검증 및 LLM 기반 자연어 피드백은 HOPE Research 후속 단계(8월~)에서 박규동 교수 협의 하에 진행 예정이다.
2.4 명시할 한계
발표 자료에 다음 한계를 명시한다.
	•	misarticulation 인식 정확도의 정량 evidence 부재 (라이브 시연으로 정성 검증)
	•	조음음운장애 아동의 비정형 음향에서의 robustness 미검증 (후속 임상 협력 필요)
	•	Module D 피드백 템플릿의 SLP 검수 미진행 (사용 시 보호자·교사 동행 권장)

4.1 Core Principle
정상 발화 데이터로만 학습한다. 합성 misarticulation, 직접 수집 데이터, 외국인 발음 데이터는 모두 사용하지 않는다.

5.5 Module A+: Forced Alignment (★ v0.8 신규)
v0.3에서 누락되었던 모듈. CTC posterior로부터 음소별 frame boundary를 결정.
import torchaudio.functional as F
 
def forced_align(audio_features, predicted_phonemes, blank_idx=0):
    """
    Args:
        audio_features: (1, T, vocab_size) log probabilities
        predicted_phonemes: (1, N) phoneme indices
    Returns:
        list of (phoneme_idx, start_frame, end_frame)
    """
    alignments, scores = F.forced_align(
        audio_features,
        predicted_phonemes,
        blank=blank_idx
    )
    
    # 연속된 같은 phoneme의 frame 범위 추출
    boundaries = []
    current_phoneme = alignments[0, 0].item()
    start = 0
    for t in range(1, alignments.shape[1]):
        if alignments[0, t].item() != current_phoneme:
            boundaries.append((current_phoneme, start, t-1))
            current_phoneme = alignments[0, t].item()
            start = t
    boundaries.append((current_phoneme, start, alignments.shape[1]-1))
    
    return boundaries
Module B-2와 Module A++ 모두 frame-음소 매핑이 필요하므로 forced alignment는 필수 전처리.
5.6 Module A++: OpenSMILE Acoustic Analysis (★ v0.8 신규)
지도교수 자문 반영. Wav2Vec2-CTC의 이산 출력 한계를 연속적 acoustic feature로 보완.

5.6.2 음소별 Feature 집계
def aggregate_phoneme_features(features_df, phoneme_boundaries, frame_rate_ms=20):
    """각 음소 구간의 OpenSMILE feature 집계 (mean, std)"""
    phoneme_features = []
    for phoneme_idx, start_frame, end_frame in phoneme_boundaries:
        start_ms = start_frame * frame_rate_ms
        end_ms = end_frame * frame_rate_ms
        segment = features_df[
            (features_df.index >= start_ms) & 
            (features_df.index <= end_ms)
        ]
        phoneme_features.append({
            'phoneme': IPA_VOCAB_INV[phoneme_idx],
            'feature_mean': segment.mean().to_dict(),
            'feature_std': segment.std().to_dict()
        })
    return phoneme_features
5.6.3 Z-score 기반 Deviation 측정
def compute_deviation_zscore(phoneme_features, normal_dist_ref):
    """음소별 feature를 정상 분포 reference와 비교하여 z-score 산출"""
    results = []
    for pf in phoneme_features:
        phoneme = pf['phoneme']
        ref = normal_dist_ref[phoneme]
        
        # 핵심 feature 4~5개에 대한 z-score 평균
        key_features = [
            'spectralFlux_sma3nz',
            'F1frequency_sma3nz',
            'F2frequency_sma3nz',
            'mfcc1_sma3nz'
        ]
        
        z_scores = []
        for f in key_features:
            mean = pf['feature_mean'][f]
            ref_mean = ref[f]['mean']
            ref_std = ref[f]['std']
            z = abs(mean - ref_mean) / (ref_std + 1e-8)
            z_scores.append(z)
        
        avg_z = sum(z_scores) / len(z_scores)
        results.append({'phoneme': phoneme, 'z_score': avg_z})
    
    return results
5.7 Module B-1: Phoneme Alignment
Needleman-Wunsch global alignment + articulatory feature distance. v0.3 결정 그대로 유지.
FEATURES = {
    's':  {'manner': 'fricative', 'place': 'alveolar', 'voice': False},
    't':  {'manner': 'stop',      'place': 'alveolar', 'voice': False},
    'tɕ': {'manner': 'affricate', 'place': 'palatal',  'voice': False},
    'l':  {'manner': 'lateral',   'place': 'alveolar', 'voice': True},
}
 
def dist(p1, p2):
    if p1 == p2:
        return 0
    f1, f2 = FEATURES[p1], FEATURES[p2]
    diff = sum(f1[k] != f2[k] for k in f1)
    return diff / len(f1)
 
# /s/ vs /t/: manner 다름 → 0.33
# /s/ vs /m/: manner, place, voice 다름 → 1.0
5.8 Module B-2: Variation Classifier
v0.3의 ML head 폐기. Rule-based + acoustic deviation hybrid로 단순화.
def classify_variation(alignment_result, acoustic_deviations, z_threshold=2.5):
    classifications = []
    for i, (tgt, act, status) in enumerate(alignment_result):
        z = acoustic_deviations.get(i)
        
        if status == 'OK':
            # 음소는 일치 → DIS 검사
            if z is not None and z > z_threshold:
                variation = 'DIS'    # 왜곡: 음소 맞지만 음향 deviation
            else:
                variation = None     # 정상
                
        elif status == 'SUB':
            variation = match_substitution_rule(tgt, act)
            # 예: STOPPING (ㅅ→ㄷ), FRICATION
            
        elif status == 'DEL':
            variation = match_deletion_rule(tgt, position=i)
            # 예: LIQUID_DEL (ㄹ 종성 생략)
            
        elif status == 'INS':
            variation = None    # 첨가는 분류 대상 아님
        
        classifications.append((tgt, act, status, variation, z))
    
    return classifications
5.9 Module C: PCC Calculation
표준 공식 그대로. v0.3 결정 유지.
def pcc(classifications):
    correct = 0
    total_target_consonants = 0
    
    for tgt, act, status, variation, z in classifications:
        if is_consonant(tgt):
            total_target_consonants += 1
            if status == 'OK' and variation != 'DIS':
                correct += 1
    
    if total_target_consonants == 0:
        return None
    return correct / total_target_consonants * 100
 
# 주의:
# - 모음은 PCC에서 제외 (언어병리학 표준)
# - INS(첨가)는 분모에 포함 안 함
# - DIS(왜곡)는 'OK 아님'으로 분류
5.10 Module D: Feedback Generation
템플릿 기반. SLP 검수는 후속 단계에서. v0.8에서는 acoustic deviation 정량 정보를 피드백에 포함.
TEMPLATES = {
    ('STOPPING', 's'):
        "사 발음할 때 혀를 살짝 안쪽으로 넣어볼까? 바람이 새는 소리처럼.",
    ('STOPPING', 'tɕ'):
        "자 발음할 때 혀끝을 입천장에 살짝 붙여볼까?",
    ('LIQUID_DEL', 'l'):
        "끝에 'ㄹ' 소리를 빼먹었어. 혀끝을 입천장에 살짝 닿게 해볼까?",
    ('DIS', 's'):
        "사 발음에서 마찰이 약했어 (정상 대비 {z:.1f}σ).",
}
 
def generate_feedback(classifications):
    feedbacks = []
    for tgt, act, status, variation, z in classifications:
        if variation:
            key = (variation, tgt)
            if key in TEMPLATES:
                msg = TEMPLATES[key].format(z=z) if z else TEMPLATES[key]
                feedbacks.append(msg)
    return feedbacks

# src/speech_coach/serving/pipeline.py
from .schemas import AnalyzeRequest, AnalyzeResponse, PhonemeResult
from ..models import (
    PhonemeRecognizer, ForcedAligner, OpenSmileAnalyzer,
    NeedlemanWunschAligner, VariationClassifier,
    calc_pcc, FeedbackGenerator
)
from ..utils.audio import preprocess
import time
 
class InferencePipeline:
    def __init__(self, ckpt_dir, model_version, normal_dist_path):
        self.recognizer = PhonemeRecognizer.from_pretrained(ckpt_dir)
        self.forced_aligner = ForcedAligner()                  # ★ v0.8
        self.opensmile = OpenSmileAnalyzer(normal_dist_path)   # ★ v0.8
        self.nw_aligner = NeedlemanWunschAligner()
        self.var_clf = VariationClassifier()
        self.feedback = FeedbackGenerator()
        self.model_version = model_version
 
    def __call__(self, req: AnalyzeRequest) -> AnalyzeResponse:
        t0 = time.time()
        
        # 1. Preprocess
        audio = preprocess(req.audio_bytes, target_sr=16000)
        
        # 2. Phoneme Recognition (Module A)
        actual_phonemes, frame_logits, conf = self.recognizer(audio)
        
        # 3. Forced Alignment (Module A+) ★ v0.8
        phoneme_boundaries = self.forced_aligner(
            frame_logits, actual_phonemes
        )
        
        # 4. OpenSMILE Analysis (Module A++) ★ v0.8
        acoustic_deviations = self.opensmile.analyze(
            audio, phoneme_boundaries
        )
        
        # 5. Phoneme Alignment (Module B-1)
        alignment = self.nw_aligner.align(
            target=req.target_phonemes,
            predicted=actual_phonemes
        )
        
        # 6. Variation Classification (Module B-2)
        classifications = self.var_clf.classify(
            alignment, acoustic_deviations
        )
        
        # 7. PCC Calculation (Module C)
        pcc_score = calc_pcc(classifications)
        
        # 8. Feedback (Module D)
        fb = self.feedback.generate(
            req.target_word, classifications, pcc_score
        )
        
        results = [
            PhonemeResult(
                target=tgt, actual=act, status=status,
                variation=variation, acoustic_deviation_z=z
            )
            for tgt, act, status, variation, z in classifications
        ]
        
        return AnalyzeResponse(
            pcc=pcc_score,
            phoneme_results=results,
            feedback=fb,
            latency_ms=int((time.time() - t0) * 1000),
            model_version=self.model_version
        )

11.1 v0.3에서 그대로 유지된 핵심 결정
v0.3에서 정확하게 결정되어 v0.8까지 변경 없이 유지된 항목:
	•	Whisper 배제, Wav2Vec2-XLSR-Phoneme 채택
	•	CTC loss + LM-light 출력
	•	IPA 기반 vocab (35 token)
	•	Articulatory feature distance
	•	PCC 표준 공식 (모음 제외, INS 분모 제외)
	•	Speaker-disjoint split
	•	Sensitivity/Specificity framing (false alarm을 가장 해로운 오류로 정의)
	•	Stage-gated entry/exit 조건 구조
이 결정들은 도메인 통찰에서 나온 것이라 자원 환경과 무관하게 옳다.

12. 후속 과제 (HOPE Research)
본 대회 범위에서 분리된 항목들. 8월 이후 박규동 교수 협의 하에 진행 예정.
12.1 임상 검증
	•	실제 조음음운장애 아동 모집 (사설 언어치료실 협력 기관 확보)
	•	부모 동의서·익명화 정책·IRB 면제 자문 진행
	•	SLP 자문 확보 후 ground truth 라벨링
	•	PCC MAE, Sensitivity/Specificity, Pearson r 등 임상 지표 측정
	•	사전-사후 PCC 변화 통계 검증
12.2 LLM 피드백 통합
	•	gemma-2-2b-it 또는 ko-gemma fine-tune
	•	발화 분석 결과 JSON + persona prompt → kid-friendly text 생성
	•	의료 조언 회피 가드, 부모/치료사 escalation cue 제공
	•	SLP 5명이 100건 출력에 대해 적절성 평가
12.3 학술 출판
	•	한국언어치료학회(JSLHD) 또는 한국HCI학회 논문 투고
	•	기여: 한국어 IPA 기반 Wav2Vec2 fine-tuning + 정상 분포 reference 기반 deviation 측정
	•	프로젝트 코드 GitHub 공개 (논문 출판 후)
12.4 시스템 확장
	•	음소 확장: MVP 3종 → 전체 한국어 자음
	•	대상 확장: 5~7세 영유아, 13세+ 청소년, 다문화 아동
	•	부모/치료사 대시보드 v2
	•	on-device 추론 (ONNX + INT8 quantization)
